# Анализатор NGINX лог-файлов
Упрощает анализ NGINX лог-файлов.

## Функциональные требования
* Программа должна принимать на вход путь к лог-файлам, который может быть в виде шаблона для локальных файлов (https://en.wikipedia.org/wiki/Glob_(programming)) или URL;
* Программа должна поддерживать опциональные параметры: from, to - для анализа с фильтрацией по времени; выходной формат данных в виде markdown или adoc-документа;
* Программа должна подсчитывать общее кол-во запросов;
* Программа должна определять наиболее часто запрашиваемые ресурсы;
* Программа должна определять наиболее часто встречающиеся коды ответа;
* Программа должна рассчитывать средний размер ответа сервера;
* Программа должна рассчитывать 95-й, 99-й процентили размера ответа сервера;
* Программа должна поддерживать фильтрацию - анализ только определенных логов. Это должно быть реализовано с помощью параметров `filter-type`, `filter-value`.

## Нефункциональные требования
* Программа должна обрабатывать лог-файлы поэтапно, не загружая весь файл в память.

## Входные данные
* Путь к лог-файлам: локальный с поддержкой glob или URL;
* Временные параметры 'from', 'to' в формате ISO8601;
* Формат вывода 'markdown', 'adoc'.

## Выходные данные
* Текстовый отчет в выбранном формате, который сохраняется в директорию results.

## Инструкция по запуску
Выполнить: `go run .\cmd\run\main.go --help`
